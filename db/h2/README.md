# База данных мессенджера

## Описание базы данных

Эта база данных PostgreSQL предназначена для хранения данных мессенджера. 

## Структура таблиц

### Основные сущности

1. **Пользователи (messenger.users)**
    - Уникальные идентификаторы: username, phone, email
    - Личная информация: first_name, last_name
    - Статус и активность: status, last_seen
    - Безопасность: password_hash, password_changed_at
    - Настройки: theme, language
    - Поддержка мягкого удаления: deleted_at

2. **Чаты (messenger.chats)**
    - Типы чатов: private, group, channel
    - Метаданные: title, description, avatar_url
    - Настройки: is_public, max_members
    - Архивация: archived_at
    - Создатель: created_by

3. **Участники чатов (messenger.chat_members)**
    - Роли: owner, admin, member, guest
    - Уведомления: notifications_enabled, mute_until
    - Активность: last_read_at, unread_count
    - Статус участия: left_at

4. **Сообщения (messenger.messages)**
    - Типы сообщений: text, image, video, audio, file, sticker, poll, location
    - Структура: reply_to_id, is_edited, is_pinned
    - Партиционирование по дате создания
    - Мягкое удаление: is_deleted

5. **Реакции на сообщения (messenger.message_reactions)**
    - Эмодзи-реакции пользователей
    - Уникальность: один пользователь - одна реакция на сообщение

6. **Вложения (messenger.attachments)**
    - Метаданные файлов: file_name, file_size, mime_type
    - Ссылки на хранилище: storage_url, thumbnail_url
    - Медиа-параметры: duration, width, height
    - Время жизни: expires_at

## Пользовательские домены

База использует строгие проверки данных через домены:
- **email_domain** - валидация email адресов
- **phone_domain** - международный формат телефонов
- **username_domain** - безопасные имена пользователей
- **password_hash_domain** - поддержка современных алгоритмов хэширования
- **language_code** - стандартные языковые коды
- **theme_domain** - предопределенные темы оформления
- **mime_type_domain** - валидные MIME-типы файлов

## Основные функции

### Управление партициями

- **create_message_partition()** - создание годовых партиций для сообщений
- **init_partitions()** - инициализация партиций на несколько лет вперед
- **cleanup_old_messages()** - очистка старых сообщений с batch-обработкой

### Бизнес-логика

- **create_private_chat()** - создание приватного чата между двумя пользователями с проверками
- **increment_unread_count()** - увеличение счетчиков непрочитанных сообщений

### Автоматизация

- **update_updated_at()** - автоматическое обновление временных меток
- **messages_partition_trigger()** - автоматическая маршрутизация сообщений в партиции

## Представления (Views)

1. **v_partitions** - мониторинг состояния партиций (размеры, количество строк)
2. **v_chat_stats** - статистика по чатам (участники, сообщения, активность)
3. **v_user_stats** - статистика по пользователям (чаты, сообщения, вложения)

## Запуск через Docker

### Требования

- Docker и Docker Compose
- Файл .env с переменными окружения

### Конфигурация .env

```env
POSTGRES_DB=messenger
POSTGRES_USER=messenger_user
POSTGRES_PASSWORD=secure_password
POSTGRES_PORT=5432
```

### Запуск

```bash
docker-compose up -d
```

### Инициализация базы данных

При первом запуске автоматически выполняются:
1. Создание схемы messenger и установка расширений
2. Определение пользовательских доменов и типов
3. Создание всех таблиц и индексов
4. Определение функций и триггеров
5. Создание представлений
6. Заполнение тестовыми данными (пользователи, чаты, сообщения)

### Доступ к базе данных

- Хост: localhost
- Порт: указанный в POSTGRES_PORT (по умолчанию 5432)
- База данных: messenger
- Пользователь: messenger_user

### Остановка

```bash
docker-compose down
```

### Просмотр логов

```bash
docker-compose logs -f postgres
```

## Особенности реализации

1. **Производительность**

    - Партиционирование сообщений по годам
    - Оптимизированные индексы для частых запросов
    - Batch-обработка для очистки данных

2. **Безопасность**

    - Валидация данных на уровне базы
    - Поддержка современных алгоритмов хэширования паролей
    - Защита от SQL-инъекций через параметризованные запросы

3. **Масштабируемость**

    - Поддержка горизонтального масштабирования через партиционирование
    - Эффективные foreign keys с каскадным удалением
    - Возможность архивации старых данных

4. **Целостность данных**

    - Транзакционная обработка
    - Проверки ограничений на уровне базы
    - Консистентность через триггеры и функции
