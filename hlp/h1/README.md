# Банковская система - консольное приложение

Консольное приложение для моделирования работы банковской системы. Реализует создание клиентов, управление счетами, проведение операций и формирование отчетов.

## Структура проекта

```
src/
├── Main.java
├── bank/
│   ├── BankService.java
│   ├── impl/
│   │   └── Bank.java
│   └── console/
│       └── BankConsoleMenu.java
├── logger/
│   └── Logger.java
├── model/
│   ├── Customer.java
│   └── account/
│       ├── Account.java
│       ├── AccountType.java
│       └── impl/
│           ├── DebitAccount.java
│           └── CreditAccount.java
└── transaction/
    ├── Transaction.java
    └── TransactionType.java
```

## Компиляция

1. Откройте командную строку в корневой директории проекта
2. Выполните команду компиляции:

```bash
javac -d build -sourcepath src src/Main.java
```

Альтернативный способ для Windows (если предыдущий не работает):

```bash
javac -d build src\Main.java src\bank\BankService.java src\bank\impl\Bank.java src\bank\console\BankConsoleMenu.java src\logger\Logger.java src\model\Customer.java src\model\account\Account.java src\model\account\AccountType.java src\model\account\impl\DebitAccount.java src\model\account\impl\CreditAccount.java src\transaction\Transaction.java src\transaction\TransactionType.java
```

## Запуск

После успешной компиляции выполните:

```bash
java -cp build Main
```

## Функциональность

### Основные возможности

1. **Управление клиентами**
    - Создание нового клиента с уникальным ID
    - Поиск клиента по ID

2. **Типы счетов**
    - Дебетовый счет: баланс не может быть отрицательным
    - Кредитный счет: баланс может уходить в минус до установленного лимита

3. **Банковские операции**
    - Пополнение счета
    - Снятие средств
    - Перевод между счетами
    - Валидация всех операций

4. **Отчетность**
    - Просмотр счетов клиента
    - История всех транзакций
    - Статистика банка

### Меню приложения

```
===== Банковская система =====
1. Создать клиента
2. Открыть дебетовый счет
3. Открыть кредитный счет
4. Пополнить
5. Снять
6. Перевести
7. Показать счета клиента
8. Показать транзакции
9. Отчет банка
10. Выход
```

## Особенности реализации

### Архитектура

- **Интерфейс BankService** - абстракция банковских операций
- **Класс Bank** - основная реализация бизнес-логики
- **Account** - абстрактный базовый класс для счетов
- **DebitAccount/CreditAccount** - конкретные реализации типов счетов
- **Transaction** - неизменяемая запись для хранения информации об операциях

### Управление состоянием

- Статический счетчик ID клиентов гарантирует уникальность
- Все операции логируются в список транзакций
- Состояние хранится в памяти (не персистентное)

## Валидация

1. **Суммы операций**: должны быть больше 0
2. **Дебетовые счета**: нельзя снять больше текущего баланса
3. **Кредитные счета**: нельзя превысить кредитный лимит
4. **Поиск**: клиенты и счета должны существовать для операций
