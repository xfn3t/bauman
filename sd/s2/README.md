# Система управления спутниковой группировкой

Проект представляет собой объектно-ориентированную систему для управления группировкой искусственных спутников Земли.

## Компиляция и запуск

Запуск производится из корня проекта, директории s2

### 1. Через Gradle (рекомендуется)

```bash
./gradlew build
./gradlew run
```

#### Запуск тестов

```bash
./gradlew test
```

### 2. Через Java (ручная компиляция)

```bash
javac -d app/build/classes/java/main app/src/main/java/ru/bmstu/sd/s2/App.java app/src/main/java/ru/bmstu/sd/s1/satellite/*.java

java -cp "app/build/classes/java/main" ru.bmstu.sd.s2.App
```

## О проекте

Система моделирует управление различными типами спутников:
- Спутники связи (CommunicationSatellite) для передачи данных
- Спутники дистанционного зондирования (ImagingSatellite) для съемки территории
- Управление группировками спутников (SatelliteConstellation)

### Архитектура системы

Проект реализован с соблюдением принципов SOLID:

1. **EnergySystem** - отвечает исключительно за управление энергией спутника
2. **SatelliteState** - отвечает исключительно за управление состоянием спутника
3. **Satellite** - абстрактный базовый класс, использующий композицию для делегирования обязанностей
4. **CommunicationSatellite/ImagingSatellite** - конкретные реализации специализированных спутников
5. **SatelliteConstellation** - класс для управления группой спутников

### Основные функции:

- Активация/деактивация спутников с проверкой уровня заряда
- Выполнение специализированных миссий для каждого типа спутника
- Управление энергопотреблением и автоматический контроль уровня заряда
- Полиморфное выполнение задач через общий интерфейс
- Формирование и управление спутниковыми группировками

### Технические требования:

- Java 17 или выше
- Для сборки через Gradle: JDK с поддержкой Java 17

### Особенности реализации:

- Принцип единственной ответственности (SRP): каждый класс имеет одну причину для изменения
- Принцип открытости/закрытости (OCP): система легко расширяется новыми типами спутников
- Использование композиции вместо наследования для управления энергией и состоянием
- Полное покрытие модульными и интеграционными тестами
- Корректная обработка граничных условий и ошибок
